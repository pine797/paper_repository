## 状态变量识别过程

1.获取消息处理函数的入口（动态）
    备注：GDB自动化脚本

2.提取消息处理函数（静态）pass1
    输入：GDB的函数栈信息
    输出：协议消息处理函数的范围

3.提取候选的米利机变量（静态）  pass2
    输入：协议消息处理函数的范围
    输出：候选米利机变量的具体信息

4.识别正式的米利机变量（动态）
    输入：候选米利机变量的具体信息
    输出：变量名称与函数栈 => 正式的米利机变量信息（类型、具体代码位置）、子状态变量信息

5.识别并插桩状态变量（静态）    pass3
    1.输入：正式的米利机变量信息（类型、具体代码位置）、子状态变量信息；
            关键字字典
    2.处理：识别子状态变量、插桩
    3.输出：插桩的变量信息、llvm ir

## pass设计

* pass1暂时单独一个，后期合并进来
* pass2和pass3合并起来，根据选项确定功能分支
